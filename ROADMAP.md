### [x] Hybrid Retrieval

Configurable policies: Parallel, VectorThenGraph, GraphThenVector with weighted merging

<a id="vector-retrieval"></a>

### [x] Vector Retrieval

Index, BatchIndex, IndexManager, Embedder interfaces with VectorRetriever implementation

<a id="pgvector"></a>

### [x] pgvector Provider

<a id="memory-implementations"></a>

VectorIndex, KnowledgeGraph, HashEmbedder for testing and development

<a id="reranking"></a>

### [x] Reranking Support

CrossEncoder, Heuristic (RRF, min-score), and Chain rerankers

**Version:** 0.1.0 (2026-01-11)

## Design Principles <a href="#omniretrieve-roadmap">â†‘ Top</a>

- **Unified Interface** - Single Retriever interface works with any backend
- **Strategy Flexibility** - Support vector, graph, and hybrid retrieval
- **Backend Agnostic** - Swap providers without changing application code
- **Observable** - Built-in tracing for debugging and optimization
